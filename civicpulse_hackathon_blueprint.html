Product vision
What makes CivicPulse stand out from every other civic reporting app

Core problem

Citizens have no way to report hyperlocal issues — potholes, broken streetlights, water leakage — in a way that guarantees action. Existing tools are either government portals nobody checks, or social media posts that get ignored. The gap: AI-powered closed-loop accountability that takes a photo and autonomously routes it to the right authority, validates it with community consensus, predicts recurrence, and rewards civic participation.

Target users

  • Citizens (reporters) — 80% of traffic
  • Community moderators — trusted validators
  • Municipal authorities — resolution owners
  • Ward councillors — accountability layer
  • NGOs / civic tech orgs — bulk monitoring

Why judges will love it

  • Multi-agent Gemini system (not just one API call)
  • Predictive analytics beyond reactive reporting
  • Structured outputs + function calling throughout
  • Firebase real-time + Google Maps deep integration
  • Gamification loop that increases engagement metrics

Competitive differentiators

  • Autonomous duplicate detection — no manual triaging
  • Severity scoring with SLA auto-assignment
  • Predictive heatmaps (ML over historical data)
  • Resolution recommendation with cost estimation
  • Community consensus before escalation (anti-spam)

The one-liner pitch

"CivicPulse turns a citizen's photo into an autonomous chain of AI agents that validate, classify, route, and predict infrastructure failures — making every street problem impossible to ignore."

Multi-agent AI architecture
9 Gemini-powered agents collaborating autonomously via function calling
1. Vision Analysis Agent

Input: Image bytes + GPS · Output: Structured JSON {issue_detected, confidence, visual_description, damage_extent} · Model: gemini-2.5-pro with vision · Key: Uses structured output schema to guarantee parseable JSON, never freeform text

2. Categorization Agent

Input: Vision output + description · Output: {category, sub_category, tags[]} from enum set · Categories: ROAD_DAMAGE, WATER_INFRASTRUCTURE, ELECTRICAL, SANITATION, PUBLIC_SAFETY, ENCROACHMENT · Uses function calling to query historical category distribution

3. Severity Scoring Agent

Input: Category + visual description + location · Output: {severity_score: 1-10, urgency: LOW/MEDIUM/HIGH/CRITICAL, sla_hours, risk_factors[]} · Logic: Weights safety risk, population density, recurrence history · SLA auto-assigned based on score

4. Duplicate Detection Agent

Input: GPS coordinates + category + image embedding · Output: {is_duplicate, matched_report_id, similarity_score} · Method: Geo-radius query (50m) + semantic similarity on descriptions + image hash comparison · Merges duplicates, credits original reporter

5. Routing Agent

Input: Category + ward/zone from Maps API + severity · Output: {authority_id, department, escalation_path[], estimated_resolution_days} · Uses function call to query authority_zones Firestore collection · Cascading escalation if SLA breached

6. Community Engagement Agent

Input: New report + nearby user list · Output: FCM notification payload + verification prompts · Personalizes message based on user's past engagement history · Awards XP for verifications

7. Resolution Recommendation Agent

Input: Issue category + severity + historical resolutions · Output: {recommended_action, estimated_cost_inr, required_resources[], reference_cases[]} · Generates authority action letter as markdown · Pulls similar resolved cases as references

8. Predictive Analytics Agent

Input: Historical reports + seasonal data + urban density · Output: {risk_zones[], predicted_issues[], prevention_recommendations[]} · Runs weekly batch · Outputs GeoJSON heatmap data for Maps overlay · Biggest "wow" factor for judges

9. Orchestrator Agent (Coordinator)

Role: Master controller using Gemini function calling to sequence agents 1-7 for each new report · Maintains agent state log visible in UI (show judges the chain!) · Handles failures with retry + fallback · All agent results written to Firestore for auditability

Sample Gemini prompt — Vision Analysis Agent

You are a civic infrastructure analysis agent. Analyze the provided image and return ONLY valid JSON matching this schema: { "issue_detected": boolean, "confidence": float (0-1), "issue_type": string, "visual_description": string (max 100 words), "damage_extent": "MINOR" | "MODERATE" | "SEVERE" | "CRITICAL", "safety_risk": boolean, "affected_area_sqm": number | null } Image context: GPS lat={lat}, lng={lng}, timestamp={ts} Do not include any text outside the JSON object.

Sample Gemini prompt — Severity Scoring Agent

You are a municipal severity assessment agent. Given this civic issue, output a JSON severity assessment: Issue: {visual_description} Category: {category} Location type: {location_type} (residential/commercial/highway) Historical reports at this location: {count} Return JSON: { "severity_score": integer (1-10), "urgency": "LOW" | "MEDIUM" | "HIGH" | "CRITICAL", "sla_hours": integer, "risk_factors": string[], "public_safety_threat": boolean }

Gamification system
Designed to increase DAU, retention, and community verification rates

XP system

  • Submit a new report: +25 XP
  • Report gets verified: +15 XP
  • Report gets resolved: +50 XP
  • Verify another report (correct): +10 XP
  • First report in ward: +100 XP bonus
  • Streak (7 days active): +75 XP
  • High-severity report: XP ×2 multiplier

Levels

  • Level 1 — Watchful Citizen (0–100 XP)
  • Level 2 — Street Guardian (100–300 XP)
  • Level 3 — Ward Champion (300–700 XP)
  • Level 4 — Civic Hero (700–1500 XP)
  • Level 5 — Community Legend (1500+ XP)

Higher levels → moderator privileges → authority of early access

Badges (judge-visible)

  • 🔭 First Spotter — first report in area
  • ⚡ Fast Responder — verified within 1hr
  • 🎯 10× Reporter — 10 confirmed reports
  • 🌧️ Monsoon Guard — reported water issues
  • 🌙 Night Watch — reported after 10pm
  • 🏆 Top Ward — #1 in ward leaderboard

Leaderboards

  • Ward leaderboard (most relevant — hyperlocal)
  • City leaderboard (weekly reset)
  • Category specialists (pothole hunters, etc.)
  • Authority response rate ranking (accountability!)

Authority leaderboard is unique — creates public pressure to resolve issues fast
